Configuration Files

The VHQ control application relies on configuration files to determine which modules to control and what their initial settings should be. The configuration file is a Tcl script. The configuration file sets Tcl variables that are interpreted both as device identification/connection information and as settings. This page describes the variables you can set.

Identification/connection variables

Identification and configuration variables supply information used by people and computers to locate the module:

  • name provides an english name for the device.
  • crate is the number of the VME crate the device is installed in.
  • base is the base address of the device in the VME crate.
  • maxv is the maximum voltage capability of the module
  • maxi is the maxium current that can be sourced by the module
All of these variables are required to have values in configuration files. If a configuration file is used as a settings file, these values will be ignored and can be omitted.

Settings variables

Settings variables represent module settings. When settings are loaded, the application never performs a ramp to a target. Voltage ramps can only be initiated manually.

The settings variables are:

  • Ilimit(chan) Defines the soft current limit for channel chan. This limit is expressed in uA.
  • RampSpeed(chan) Defines the ramp speed for channel chan. This speed is in units of Volts/sec.
  • SetPoint(chan) Defines the ramp target voltage for channel chan. This is in units of Volts.
